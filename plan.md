# Plan Document

## 1. Repository Plan

### Goals
- Create and maintain a public GitHub repository for HW 4.4
- Show evidence of branching, issues, commits, and pull requests
- Keep the `main` branch updated after merging work from the development branch

### Needs
- A public GitHub repository
- A `main` branch and at least one additional branch
- At least two GitHub issues
- Meaningful commit messages
- A pull request that merges changes into `main`

### Steps
1. Create the repository and confirm that it is public.
2. Create at least two issues for the required tasks.
3. Create a development branch from `main`.
4. Make all edits on the development branch.
5. Commit changes with clear messages that reference the issues.
6. Open a pull request and merge it into `main`.
